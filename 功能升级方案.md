## 第一部分：国际化（i18n）多语言升级方案

### 1.1 现状评估

项目已具备 i18n 基础框架：

| 组件 | 状态 | 说明 |
|------|------|------|
| `js/i18n.js` | 已存在 | 768行，含 `t()` / `setLang()` / `updatePage()` / `init()` 方法 |
| `data-i18n` 属性 | 已部署 | 所有 HTML 页面中关键文本节点已添加标记 |
| 语言切换 UI | 已存在 | 导航栏含 EN / 中文切换按钮 |
| 语言持久化 | 已实现 | 通过 `localStorage.setItem('waterbearintl_lang', lang)` |
| 自动检测国家 | **缺失** | 当前仅读取 localStorage，无 IP 定位逻辑 |
| 英文翻译完整性 | **有缺陷** | en 翻译中混入大量未翻译的中文字符串（详见核查报告） |

**结论：框架已有，但需修复数据 + 增加自动检测能力。**

### 1.2 技术选型

#### 方案：IP 地理位置 API + 前端 i18n 增强

| 层次 | 选型 | 理由 |
|------|------|------|
| **IP 定位** | ipapi.co / ipinfo.io 免费 API | 免费额度充足（1000次/天），无需后端改动 |
| **语言映射** | 前端逻辑 | 根据国家代码映射到语言：CN→zh, 其他→en |
| **翻译管理** | 现有 `i18n.js` 增强 | 最小改动，修复数据即可 |
| **国际化路由** | URL 参数 + cookie | `?lang=zh` + `set-cookie`，SEO 友好 |
| **hreflang** | HTML `<link>` 标签 | Google 多语言 SEO 必需 |

#### 为什么不选其他方案？

| 方案 | 不选理由 |
|------|----------|
| i18next 等第三方库 | 引入新依赖但现有框架已能覆盖需求；增加约 40KB 体积 |
| 后端语言检测（Accept-Language） | 仅依赖浏览器语言头，无法区分中国 vs 海外中文用户 |
| Google Translate 插件 | 翻译质量不可控、SEO 无效、有广告 |
| 纯 URL 子目录 (`/en/` `/zh/`) | 需要服务器 URL 重写，改动过大 |

#### Phase 2：添加 IP 自动检测（2 小时）

在 `i18n.init()` 中新增逻辑：

```javascript
// i18n.js 新增方法
async detectCountryAndSetLang() {
  const savedLang = localStorage.getItem('waterbearintl_lang');
  if (savedLang) return savedLang;  // 用户手动选择优先

  try {
    // 使用 ipapi.co 免费 API（无需 API key，1000次/天）
    const res = await fetch('https://ipapi.co/json/', { signal: AbortSignal.timeout(3000) });
    if (!res.ok) throw new Error('IP lookup failed');
    const data = await res.json();

    // 中国大陆 → 中文，其他 → 英文
    return data.country_code === 'CN' ? 'zh' : 'en';
  } catch (e) {
    console.warn('IP geolocation failed, defaulting to en:', e.message);
    return 'en';  // 降级：默认英文
  }
},

// init() 改为 async
async init() {
  const lang = await this.detectCountryAndSetLang();
  this.setLang(lang);
  // ... 原有事件绑定
}
```

**降级策略（多层级）：**

```
1. localStorage 用户手动选择（最高优先级）
2. IP 地理定位 API
3. navigator.language 浏览器语言头
4. 默认 'en'
```

#### Phase 3：SEO 增强 — hreflang 标签（1 小时）

在每个 HTML `<head>` 中动态插入：

```html
<link rel="alternate" hreflang="en" href="https://waterbearintl.com/">
<link rel="alternate" hreflang="zh" href="https://waterbearintl.com/?lang=zh">
<link rel="alternate" hreflang="x-default" href="https://waterbearintl.com/">
```

由 `i18n.js` 在 `init()` 阶段动态生成：

```javascript
function injectHreflang() {
  const base = window.location.origin + window.location.pathname;
  const links = [
    { hreflang: 'en', href: base + '?lang=en' },
    { hreflang: 'zh', href: base + '?lang=zh' },
    { hreflang: 'x-default', href: base }
  ];
  links.forEach(l => {
    const link = document.createElement('link');
    link.rel = 'alternate';
    link.hreflang = l.hreflang;
    link.href = l.href;
    document.head.appendChild(link);
  });
}
```

## 第二部分：国际支付集成方案

### 2.1 现状评估

**当前项目无任何支付功能。** 前端仅有询盘表单（联系表单），商业模式为线下报价+线下交易（传统 B2B 外贸模式）。

### 2.2 B2B 外贸场景分析

| 场景 | 支付需求 | 推荐方案 |
|------|----------|----------|
| **样品费 / 小额定金** | 在线支付 $50~$2000 | PayPal / Stripe |
| **大额订单** | 信用证 / T/T 电汇 | 线下处理（表单收集需求） |
| **定期采购客户** | 在线付款 + 账单 | Stripe Invoice |
| **展会/线上获客** | 快速支付样品费 | PayPal 一键付 |

**核心思路：不改造 B2B 大额交易流程，仅在网站增加"在线支付样品费/定金"能力，降低获客摩擦。**

### 2.3 技术选型

| 支付渠道 | 覆盖地区 | 接入方式 | 手续费 | 推荐度 |
|----------|----------|----------|--------|--------|
| **PayPal** | 全球 200+ 国家 | PayPal JS SDK（前端按钮） | 4.4% + 固定费 | ⭐⭐⭐⭐⭐ |
| **Stripe** | 46 个国家 | Stripe Checkout / Elements | 2.9% + $0.30 | ⭐⭐⭐⭐ |
| **Payoneer** | 全球（B2B 专长） | API 集成 | 1%~2% | ⭐⭐⭐ |
| **Alipay+** | 亚洲市场 | API | 约 2% | ⭐⭐⭐ |

**推荐组合：PayPal（主）+ Stripe（备），理由：**

1. PayPal 在欧美外贸场景认知度最高，买家信任
2. Stripe 费率更低，适合做 Stripe Invoice（发送含支付链接的账单邮件）
3. 两者均支持 Webhook 异步通知，无需轮询订单状态
4. 均提供 Sandbox 测试环境

```javascript
// js/payment.js
paypal.Buttons({
  createOrder: async () => {
    const res = await fetch('/api/payment/create-order', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ amount: 150.00, currency: 'USD', sku: 'CNC-800M' })
    });
    const data = await res.json();
    return data.orderID;
  },
  onApprove: async (data) => {
    const res = await fetch('/api/payment/capture-order', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ orderID: data.orderID })
    });
    const result = await res.json();
    if (result.status === 'COMPLETED') {
      // 跳转到支付成功页
      window.location.href = `/payment-success.html?order=${result.order_id}`;
    }
  },
  onError: (err) => {
    console.error('PayPal error:', err);
    showNotification('Payment failed. Please try again.', 'error');
  }
}).render('#paypal-button-container');
```

**2.2 Stripe 支付链接方案**（更轻量，无需前端 JS SDK）：

后端生成 Stripe Payment Link，通过邮件发送给客户。适合 B2B 场景中"先询盘后付款"的流程：

```javascript
// server.js 新增
app.post('/api/payment/stripe-invoice', async (req, res) => {
  const session = await stripe.checkout.sessions.create({
    payment_method_types: ['card'],
    line_items: [{
      price_data: {
        currency: 'usd',
        product_data: { name: 'Sample Fee - CNC-800M' },
        unit_amount: 15000  // $150.00 in cents
      },
      quantity: 1
    }],
    mode: 'payment',
    success_url: `${BASE_URL}/payment-success?session={CHECKOUT_SESSION_ID}`,
    cancel_url: `${BASE_URL}/payment-cancel`,
    customer_email: req.body.email
  });
  res.json({ url: session.url });
});
```
